git에서는 여러가지 방법으로 자격증명을 할 수 있다.
여러가지 방법으로 다 보았지만 결국 하나의 플랫폼에서 멀티 계정 또는 멀티 플랫폼에 멀티 계정으로 구성되어 있는 경우 확장성과 함께 별도의 라이브러리 설치없이 설정 할 수 있는 방법은
ssh 방식으로 깃을 클론하고 플랫폼 또는 계정마다 별도의 ssh키를 만들어 설정하는 것이다.

하나의 로컬 ssh 클라이언트에 멀티 자격증명 파일을 설정하는 방법은 아래와 같다

```bash
Host github-user1
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_user1_ed25519

Host github-user2
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_user2_ed25519
```

이렇게 설정하고 사용할때에는 아래와 같이 활용 할 수 있다. 결국 플랫폼 또는 유저 별로 다른 ssh키를 사용 할 수 있게 하므로서 로컬 내에서 여러 자격 증명을 사용 할 수 있다.
주의해야할 점을 인증파일의 소유권을 600으로 설정해야 git에서 정상적으로 인증파일을 사용 할 수 있다.

```bash
github-user1:user1/test.git
github-user2:user2/test.git
```
